From sequences to knowledge, improving and learning from sequence alignments

Chapter 4 Learning from sequences and alignments

As in the previous chapter, sequence alignments are rich source of information.
Pairwise: reference-based316,317 and de novo318,319 assembly, protein function prediction320, variant calling321 and structural variant detection322,323, clustering115 MSA: phylogeny324,325, protein structure prediction326 etc…

More recently focus on machine learning: to predict drug resistance (cf. Chapter 6), predict protein structure and function, annotate genes, predict protein location, GWAS…

4.1 What to learn ?

  • The first question one might have is what and how can we learn with ML ?

  • There are different learning paradigms that inform what you can learn and predict with your ML method

4.1.1 Supervised

Here we have extra knowledge -> labels and we can use his information to train the model.

4.1.1.1 Regression

Here the goal is to predict some type of meaningful continuous numerical value: like a size or a weight. Useful in lots of different fields/

  • predict a quantified resistance level of a virus to a drug (fold change)327

  • predict drug response in cancer (i.e. IC50)328, they use genome-wide features pre-extracted from sequences and not sequence directly.

  • predict angles or contact maps in protein structure -> very active field329–333 either directly from MSAs136 or from derived features like contact maps334

  • Score MSAs335

  • predicting protein fitness for in silico protein engineering336–338

  • predicting gene expression levels from SNPs (meaning we need alignments… to a reference)339

4.1.1.2 Classification

Here we predict discrete categorical values numerically encoded

  • resistant virus or not340,341 (also Chapter 6) also many studies on antimicrobial resistance342,343

  • Cellular localization of protein344

  • Secondary structure of amino acid in protein sequence345 (\(\alpha\), \(\beta\) or other)

  • localized gene expression in cells346

  • splice site detection347

  • Methylation site prediction348

  • Protein function prediction349

4.1.2 Unsupervised

  • clustering, align all pairs of sequences for clustering350, phylogenetics could be considered as a specific type of clustering351.

  • predict mutational effects352

  • Predict recombination hotspots in humans with a MaxLik approach353

  • Approximating the edit distance and bypassing the need for alignment354

  • dimensionality reduction:

    • traditionally apply PCA to some type of matrix, like distance matrix obtained from aln355 or other matrices like gene expression

    • Also can be used for clustering356–359

    • Work done to compute PCA directly on alignment360

4.1.3 Others

  • task based: end-to-end training like aligning sequences, this is harder because it requires developing a custom differentiable scoring function based on the task.

    • Predict P/P interactions361

    • microRNA target prediction362

  • self-supervised, i.e. train on a proxy task and hope the model learns important features and useful stuff to continue on fully-unsupervised.

    • Used to find useful info in disordered protein regions363

    • Protein language models:364

  • semi-supervised, combined a small amount of labelled data with large amount of unlabeled data that the models can leverage.

    • used to predict drug-protein interactions365

    • and transmembrane protein topology366

4.3 Preprocessing the alignment for machine learning

In order to do some learning we need to have the data in digestible form

Most of the methods will transform an aligned sequence into a vector, and therefore the alignment as an input matrix.

Some methods are designed to not required alignment, but I will present some anyway

4.3.1 General purpose embeddings

review367

  • Simple labeling / ordinal coding, i.e. assigning an integer to each character (A=1, C=2, G=3, T=4)

    • pb -> assigning artificial order to a purely categorical variable, used in327

  • BItwise labeling -> encode each character with \(n\) bits, for nucleotides 2bits if no gaps (A=00, C=01, G=10, T=11)368,369 for AAs 5 bits necessary explored in370

    • still a problem of order, for aas used in hashing and bloom filters …

    • Often used for hashing/indexing a sequence or compressing, but is also a used scheme in categorical variable encoding in machine learning.

  • One-Hot encoding, widely used: a variable with \(d\) levels (for DNA \(d=4\)) is transformed to a length \(d\) sparse binary vector, when the variable is equal to the \(i^{th}\) level then the resulting vector has a 1 set in the \(i^{th}\) position. (also called orthonormal371)

    • Used in Chapter 6 and other papers372,

    • one of the problems is the “curse of dimensionality” since the for a sequence of length \(n\) we get \(n\times d\) features, which can be quite large.

    • Performance between OH and Ordinal can be quite similar373 but easily interpretable (important in Bio)

    • Has been used for encoding protein sequences as early as 1988374

Figure 4.1: Example of 3 general categorical encoding schemes

  • Many general purpose categorical encodings, library in Python375

4.3.2 Biological sequence-specific embeddings

These embeddings were inspired by physico-chemical properties, many encodings in370:

  • AAIndex embedding, based on the the AAIndex database376,

    • each amino acid has a number real-valued physico-chemical properties so you can select a subset of them.

    • A subset of properties identified as informative377

    • Available in the iFeature python package378

    • Also, combine features from AAIndex with PCA to get ensemble features and use top components379

  • Groups:

    • Based on the Venn diagram defined by Taylor380, grouping AAs by 9 properties (hydrophobic, polar, small, …)

    • We can get a 9-D vector for each residue

    • An encoding based on that was used as early as 1987 to predict protein secondary structure381

    • Later 5 more groups defined382

  • BLOMAP383:

    • Non-linear projection of the BLOSUM62 substitution matrix to encode an amino acid in 5 dimensions

    • Used to predict cleavage sites of HIV-1 protease371

  • Others:

    • frequencies of amino acids -> very coarse (but low dimensional)

    • \(k\)-mer frequencies, often reffered to as \(n\)-gram encoding384, often \(n=2\) since as \(k\)/\(n\) grow the dimensions grow exponentially -> with 20 AAs and \(20^n\) dimensions per sequence.

    • OETMAP, extension of BLOMAP385

    • Codon graph encoding proposed in370, where an AA is represented by a directed graph of all the codons that make up the AA. The 4x4 adjacency matrix is converted into a length 16 vector representing a single AA.

4.3.3 Learned embeddings

  • (Variational) Auto-encoders:

    • Bottlneck in deep neural neck, task is to predict input. Add noise in the hidden layers -> remove noise or regularize to have smooth latent space and get embeddings
    • Used for ancestral sequence reconstruction386 and estimating evolutionary distances354
    • VAEs used for sequence design as well387,388

  • NLP:

    • From the field of natural language processing where very high dimensionality (470,000 words in the Merriam-Webster English dictionary389, so naive one hot is out of the question), we need other ways to transform words into sequences.

    • Word2Vec derivatives:

      • word2Vec390,391, take in a large corpus of text and learns a vector space from it. Then each word in the corpus can be assigned a vector, constraints mean that similar words have similar vectors (i.e. low distance in the vector space). And that the embeddings make sense grammatically (e.g. of the Paper \(vec(Madrid) - vec(Spain)\) should be close to \(vec(Paris)\) in the learned space.

      • Context of a word = window of \(k\) words centered around it

      • 2 ways to train it392:

        • CBOW (continuous bag of words) = predict word from context

        • skip-gram = predict context from word

        • The model is a neural network and the hidden layer corresponds to the embedding (similar to auto-encoders)

      • dna2vec393

        • Used to predict methylation sites394

      • seq2vec395

      • BioVec/ProtVec/GeneVec396

        • Seq2vec and ProtVec both used in classification397

    • Transformers / NN-based language models:

      • Also from NLP, more recent development,

        • Some have seen a lot of success like BERT398 and GPT-3399

      • BERT / transformers

      • protein language models

      • Used to embed sequences from MSAs and find homologies400

      • MSA Transformer

  • Powerful but hard to interpret what the model actually learns. i.e. “black box”

Other weirder embeddings: chaos game theory401

4.4 Conclusion

  • Alignment rich in info

  • plenty to learn with ML

  • the choice of embedding / vector representation of sequences in the alignment is not a simple task

  • Guides what you are able to learn
